# Scrum Helper 튜토리얼 — 처음 사용하는 분들을 위한 가이드

Scrum Helper를 처음 사용하는 팀원을 위한 단계별 가이드입니다. 회원가입부터 프로젝트를 마무리하는 Wrap-up까지, 실제 사용 순서대로 따라갈 수 있게 구성했습니다.

- 배포 서비스: <https://scrumhelper.madcamp-kaist.org>
- 로컬 실행 방법: [RUNBOOK.md](RUNBOOK.md)

## 한눈에 보는 흐름

```text
회원가입/로그인 → 팀 만들기(또는 가입) → 회의록 작성 → 스펙 문서 생성
→ Task 분배 → 개인 Todo → 회고 작성 → 진행률/성장 나무 확인 → End Project → Wrap-up
```

---

## 1. 회원가입과 로그인

1. 첫 화면에서 **Sign up**으로 이동해 이름, 이메일, 비밀번호를 입력합니다.
2. 가입 후 로그인하면 팀 목록(`/teams`)으로 이동합니다.
3. 로그인 상태는 JWT로 유지되며, 우측 상단의 **Log out**으로 로그아웃할 수 있습니다.

## 2. 팀 만들기 / 팀 가입하기

팀 목록 화면에서 세 가지 방법으로 팀에 들어갈 수 있습니다.

| 방법 | 사용 시점 |
|---|---|
| **팀 생성** | 새 프로젝트를 시작할 때. 팀 이름, 설명, (선택) 가입 비밀번호를 정합니다. 생성한 사람이 팀장이 됩니다. |
| **공개/비밀번호 팀 가입** | 목록에 보이는 팀에 바로 가입합니다. 비밀번호가 걸린 팀은 비밀번호를 입력해야 합니다. |
| **초대코드 가입** | 팀장에게 받은 초대코드를 입력해 가입합니다. 팀 비밀번호와 무관하게 가입됩니다. |

> 팀장은 **Settings**에서 팀 정보 수정, 비밀번호 변경, 초대코드 재발급을, **Members**에서 팀장 위임과 팀원 제거를 할 수 있습니다.

## 3. 대시보드 훑어보기

팀에 들어가면 대시보드(`/teams/:teamId`)가 홈입니다.

- **상단 KPI 타일 4개**: 활성 Task, 완료 Task, 팀원 수, 작성된 회고 수.
- **Project progress**: 전체 Task 대비 완료율.
- **성장 나무(Growth Tree)**: Task 하나가 가지 하나입니다. 백로그는 새싹, 진행 중은 잎, 완료는 잎+열매로 표현되고, 프로젝트가 끝나기 전에는 모노톤으로 그려집니다.
- **Todo 카드**: 내가 고정해 둔 Todo Task 미리보기.
- **Quick actions**: Task 추가, 회의 기록, 회고 작성 바로가기.

## 4. 회의록 작성하기 (Meetings)

모든 흐름의 시작은 회의입니다.

1. 사이드바 **Meetings** → **회의 생성**.
2. 회의 내용을 직접 입력하거나, **녹음 파일을 업로드**하면 스크립트로 변환됩니다.
3. 저장된 회의록에서 **요약 생성**을 누르면 AI가 회의 요약을 만들어 줍니다.

## 5. 스펙 문서 만들기 (Spec)

회의에서 나온 이야기를 실행 가능한 스펙으로 정리합니다.

1. 사이드바 **Spec** → **새 스펙 문서**.
2. 참고할 **회의록을 여러 개 선택**하면 AI가 스펙 초안을 생성합니다.
3. 팀의 기준 문서로 쓸 스펙은 **Main Spec**으로 지정합니다.
4. 스펙 문서에서 **Task 추천**을 받아, 필요한 것만 골라 수락하면 바로 Task로 등록됩니다.

## 6. Task 보드 사용하기 (Task)

1. 사이드바 **Task**에서 칸반 보드(백로그 / 진행 중 / 완료)를 확인합니다.
2. **Add task**로 직접 만들거나, 스펙의 Task 추천을 수락해 채웁니다.
3. Task 상세에서 **담당자(복수 가능), 중요도, 상태**를 관리하고 **댓글**로 논의합니다.
4. Task를 완료 상태로 바꾸면 대시보드의 성장 나무에 열매가 열립니다.

## 7. 개인 Todo 관리하기 (Todo)

1. 사이드바의 Todo 목록 또는 대시보드 Todo 카드의 **Edit**으로 Todo 편집 화면(`/todos`)에 들어갑니다.
2. 내가 담당자인 Task 중 오늘 집중할 것을 고르고, **추천 Task 3개**를 참고하거나 **Add All**로 한 번에 담습니다.
3. 선택한 Todo를 바탕으로 **실행 프롬프트 생성**을 누르면, 작업을 시작할 때 쓸 수 있는 AI 프롬프트가 만들어집니다.

## 8. 회고 작성하기 (Retro)

하루를 마무리하며 회고를 남깁니다.

1. 사이드바 **Retro** → **회고 생성**.
2. **어제 한 일 / 오늘 할 일 / 궁금한 것·필요한 것·알아낸 것**을 기록합니다.
3. 함께 작업한 팀원을 **공동 작업자**로 추가하면 그 팀원도 회고를 수정할 수 있습니다.

## 9. 리더보드와 성장 나무 (Gamification)

- **Leaderboard**: 완료한 Task 수와 포인트 기준 팀 내 랭킹입니다. 포인트에 따라 명성 레벨(SEED → SPROUT → SAPLING → OAK)이 올라갑니다.
- **성장 나무**: 팀 전체의 진행 상황을 한 그루의 나무로 보여줍니다. 완료율이 65%를 넘으면 나무 꼭대기의 새순이 자라납니다.

## 10. 프로젝트 마무리 — End Project와 Wrap-up

모든 Task가 완료되면 프로젝트를 공식적으로 마무리할 수 있습니다.

1. 모든 Task가 완료되면 대시보드 우측 상단에 **END PROJECT** 버튼이 나타납니다.
2. 버튼을 누르면 **Wrap-up 리포트**로 이동합니다.
3. 첫 Wrap-up은 스크롤(또는 ↓ 키)로 한 장면씩 넘기는 **애니메이션 리포트**입니다. 스프린트 요약, 가장 활발했던 날, 최고 기여자, 최종 리더보드, 가장 길었던 회고/회의, 팀 케미까지 프로젝트의 기록을 되돌아봅니다.
4. 한 번 종료한 뒤에는:
   - 사이드바에 **Wrap-up 탭**이 생기고, 탭으로 들어가면 한 화면에 정리된 **클린 리포트**를 볼 수 있습니다.
   - 대시보드 버튼은 **SEE WRAP-UP AGAIN**으로 바뀌어 애니메이션 버전을 언제든 다시 볼 수 있습니다.
   - 성장 나무는 기본적으로 **초록색**으로 유지되며, 나무 위의 **Green / Mono 토글**로 원래 모노톤과 전환할 수 있습니다.
5. **종료 후에도 프로젝트는 계속할 수 있습니다.** 새 Task를 추가하면 대시보드에 `PROJECT ENDED · CONTINUING` 배지가 표시된 채로 작업이 이어집니다.

## 자주 묻는 질문

**Q. AI 기능(회의 요약, 스펙 초안, Task 추천, 실행 프롬프트)이 동작하지 않아요.**
Gemini API 키가 없거나 호출이 실패하면 서비스가 멈추는 대신 로컬 fallback 결과를 보여줍니다. 로컬 실행 시에는 백엔드 실행 전에 `GEMINI_API_KEY` 또는 `GOOGLE_API_KEY` 환경변수를 설정하세요.

**Q. 팀원을 제거하려는데 막혀요.**
그 팀원이 단독 담당자인 Task가 남아 있으면 제거할 수 없습니다. 먼저 해당 Task의 담당자를 재지정하세요.

**Q. END PROJECT 버튼이 안 보여요.**
완료되지 않은 Task가 하나라도 있으면 버튼이 나타나지 않습니다. Task 보드에서 남은 Task를 완료하거나 정리하세요.

**Q. Wrap-up 탭이 사이드바에 없어요.**
Wrap-up 탭은 팀에서 한 번이라도 **END PROJECT**를 누른 뒤에 생깁니다. 종료 이력은 팀 단위로 저장되어 모든 팀원에게 동일하게 보입니다.
