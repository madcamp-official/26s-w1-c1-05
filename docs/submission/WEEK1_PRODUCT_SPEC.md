# Scrum Helper Week 1 Product Spec

## 1. 프로젝트 개요

Scrum Helper는 소규모 팀 프로젝트에서 회의, 스펙 문서, Task, 개인 Todo, 회고를 한 흐름으로 연결하는 웹 기반 스크럼 관리 서비스다.

회의에서 결정한 내용이 메신저, 문서, 개인 메모에 흩어지면 실제 구현으로 이어지는 과정에서 맥락이 사라진다. Scrum Helper는 회의록을 구조화된 Spec으로 바꾸고, Spec을 실행 가능한 Task로 전환하며, 개인 Todo와 회고까지 이어지도록 설계했다.

## 2. 한 줄 소개

회의에서 나온 내용을 바로 실행 가능한 Task와 Todo로 연결하는 팀 프로젝트 운영 도구.

## 3. 개발 목적

- 웹 개발 전체 흐름을 경험한다: 기획, DB, API, 프론트, 배포, 테스트, 문서화.
- 2인 팀 프로젝트에서 필요한 협업 흐름을 실제 서비스로 구현한다.
- 회의록, 스펙, 작업, 회고가 분리되는 문제를 줄인다.
- AI 기능을 도입하되, API 실패 시에도 서비스가 동작하도록 fallback 구조를 둔다.

## 4. 예상 사용자

| 사용자 | 설명 |
|---|---|
| 팀장 | 팀을 생성하고, 팀원/팀 설정/초대코드를 관리하는 사용자 |
| 팀원 | 팀에 가입해 Task, Todo, 회의록, 회고록을 작성하는 사용자 |
| 발표/시연 참가자 | 배포 링크로 접속해 팀 프로젝트 흐름을 체험하는 사용자 |

## 5. 팀원별 역할

| 팀원 | 역할 |
|---|---|
| 안종화 | Backend API, DB schema, JWT/Auth, Gemini API, KCloud 배포, 테스트/더미 데이터 |
| 김희서 | Frontend UI/UX, React 화면 구현, 사용자 흐름 구성, 디자인 정리 |

## 6. 핵심 사용자 흐름

```text
회원가입/로그인
-> 팀 생성 또는 팀 가입
-> 회의록 작성/업로드
-> 회의록 기반 Spec 생성
-> Spec 기반 Task 추천/생성
-> 개인 Todo에 추가
-> Todo 실행 프롬프트 생성
-> Task 완료
-> 대시보드/리더보드 확인
-> 회고 작성
```

## 7. 필수 기능 명세

| 기능 | 사용자 관점 설명 | 구현 상태 |
|---|---|---|
| 회원가입/로그인 | 이름, 이메일, 비밀번호로 가입하고 로그인한다 | 완료 |
| 팀 목록 | 로그인한 사용자는 모든 팀을 볼 수 있다 | 완료 |
| 팀 생성 | 팀 이름, 설명, 선택 비밀번호로 팀을 만든다 | 완료 |
| 공개 팀 가입 | 비밀번호가 없는 팀은 즉시 가입한다 | 완료 |
| 비밀번호 팀 가입 | 비밀번호가 있는 팀은 비밀번호 입력 후 가입한다 | 완료 |
| 초대코드 가입 | 초대코드를 입력하면 비밀번호와 무관하게 가입한다 | 완료 |
| 팀장 관리 | 최초 생성자가 팀장이며, 팀장 변경이 가능하다 | 완료 |
| 팀원 관리 | 팀장은 팀원을 제거하거나 역할을 변경할 수 있다 | 완료 |
| Task 생성 | 제목, 설명, 중요도, 담당자를 입력한다 | 완료 |
| Task 상태 변경 | Task를 Backlog, In Progress, Done으로 이동한다 | 완료 |
| Task 댓글 | Task 상세에서 의견을 남기고 수정/삭제한다 | 완료 |

| 공동 작업자 | 회고록 작성자가 공동 작업자를 지정하고 권한을 관리한다 | 완료 |

## 10. 주요 정책

- 사용자는 여러 팀에 소속될 수 있다.
- 팀 이름은 unique하다.
- 팀장은 팀마다 정확히 한 명이다.
- 팀 비밀번호와 사용자 비밀번호는 해시로 저장한다.
- Task 담당자는 1명 이상이어야 한다.
- Todo에 추가된 Task는 In Progress 상태가 된다.
- Done 처리된 Task는 개인 Todo에서 제거된다.
- Gemini API 실패 또는 키 미설정 시 local fallback을 반환한다.
- Spec 생성 프롬프트는 선택한 회의록 내용에만 근거한다.
